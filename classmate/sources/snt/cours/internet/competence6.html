Les langages du Web

Pour construire un site Web, on utilise plusieurs types de langages : des langages de description tels que HTML et CSS qui sont au coeur du fonctionnement du Web et définissent la structure de chaque page visitée, des langages de programmation tels que JavaScript, PHP, Java, Python,... et un langage de traitement des bases de données tel que SQL.

Pour mieux comprendre ce que fait chaque langage, on peut simplifier les explications de la façon suivante :

  • HTML gère le contenu des pages
  • CSS gère le style des pages
  • Python, PHP, ... gère les autorisations d'accès, les affichages conditionnels, les différents traitements demandés, et l'accès aux bases de données
  • SQL gère le stockage des données et leur récupération

Chacun de ces langages va interagir avec un interlocuteur différent. Certains vont être interprétés par un client (le navigateur), d'autres par un serveur :

  • les langages interprétés par un client (on parle de langages frontend) tels que HTML, CSS et JavaScript qui décrivent comment le site Web doit s'afficher. Ils peuvent par exemple afficher un menu clignotant dans la page Web
  • les langages interprétés par un serveur (on parle de langages backend) tels que PHP, Java, Python, C# qui décrivent comment le site Web doit se comporter en appliquant des régles définies par une application. Ils peuvent par exemple n'afficher la page Web que si l'utilisateur s'est identifié. Ils réalisent l'interface avec le langage de requête de bases de données
  • les langages de communication avec une base de données tel que SQL

Attention, JavaScript est une particularité car il peut également interagir côté serveur.

Plutôt que de commencer avec une page blanche pour construire un site Web, on préfère utiliser des frameworks, qui sont des boîtes à outil pratiques, permettant de faciliter le travail. Les principaux frameworks pour le Web sont les suivants :

  • Symfony et Zend pour PHP
  • NodeJS et Angular pour JavaScript
  • Java EE pour Java
  • Django pour Python
  • ASP.NET pour C#
  • Ruby on Rails pour Ruby

On peut également utiliser un CMS Content Management System, qui est un site prêt à être modifié. Le CMS le plus populaire est actuellement WordPress,

Création d'une page Web

Il faut commencer par créer la structure de la page Web. C'est le rôle du langage HTML qui sert à gérer et organiser le contenu des pages Web. C'est donc en HTML qu'on écrit le texte, les liens hypertexte, les images et vidéos. L'information apparaît de manière brute sans mise en forme autre que structurelle.

HTML signifie HyperText Markup Language. Il s'agit d'un langage de description permettant d'écrire de l'hypertexte et construit à partir de balises pour représenter les pages Web.

Commencez donc par ouvrir un document vierge à l'aide d'un Editeur de Texte comme Sublim Text, l'enregistrer au format index.html, puis recopiez la structure suivante :

<!DOCTYPE HTML>
<html>
<head>
  <title> Titre qui apparaît dans l'onglet </title>
  <meta charset="utf-8">
</head>

<body>
  <header>
    <!-- en-tête -->
  </header>

    <p> Ceci est le corps du document </p>

  <footer>
    <!-- pied de page -->
  </footer>
</body>
</html>

Ce code est composé de :

  • un DOCTYPE qui indique au navigateur qu'il s'agit d'une page Web écrite en HTML,
  • une partie <head> qui fournit des informations générales destinées au navigateur et n'apparaissant pas sur la page lue par l'utilisateur,
  • la partie principale 6ltbody>, structurée en sections et blocs, qui comprend elle-même un en-tête <header> et un pied de page <footer>.

Afin d'exécuter ce code, il suffit simplement d'ouvrir le fichier index.html à l'aide du navigateur de votre choix Chrome, Mozilla Firefox, Safari, ...

On remarque que les commentaires sont entourés des symboles <!-- et -->.

On remarque également que le code est organisé à l'aide de balises qui permettent de donner des instructions à l'ordinateur, pour différencier langage et texte. Elles sont entourées de chevrons < et >. Ainsi, il est possible d'appliquer le même traitement à ce qui se trouve entre deux balises identiques.

On peut définir le style de chacune de ces balises séparément. Essayez par exemple de rajouter la ligne suivante entre le header et le footer :

<p style="color:blue;">hello world </p>

Bien que cela fonctionne, il serait trop fastidieux d'utiliser cette méthode pour chaque balise à modifier. On utilise plutôt une feuille de style CSS dans laquelle on mentionne les propriétés de chaque balise. Pour cela, ouvrez un second document vierge avec votre Editeur de Texte, enregistrez-le au format style.css puis recopiez le code suivant.

p {
    color: green;
}

Il faut ensuite rajouter dans la partie <head> du fichier index.html un lien vers la feuille style.css que vous venez de créer, afin que le navigateur sache comment trouver la feuille de style et la relier à la page HTML :

<link rel="stylesheet" href="style.css">

Observez maintenant ce qu'affiche votre navigateur.

Vous pouvez dès à présent rajouter un titre à l'aide des balises <h1>, <h2>, ... , puis un lien hypertexte à l'aide de la balise

<a href="adresse_du_lien">texte affiché pour le lien</a>

Il est important de ne pas croiser les balises. En HTML les erreurs sont d'autant plus difficiles à détecter que le compilateur en autorise énormément avant de bloquer. Il part du principe que ce qu'il ne comprend pas, il ne l'affiche pas, mais n'interrompt pas pour autant l'exécution du code. Par exemple le code suivant est incohérent :

<h1> mon titre <p> ici se trouve un paragraphe </h1> </p>
