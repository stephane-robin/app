Sécurité sur le Web

📚 Compétence : comprendre les différents risques

------------------------------

La cybercriminalité peut prendre différentes formes

Les principales menaces en cybercriminalité sont les suivantes :

  • vol d’information – cas particulier usurpation d’identité (spoofing)
  • modification d’information par un utilisateur non autorisé
  • déni de service DoS – cas particulier ralentissement du système (downtime)
  • répudiation (exemple : effacer la preuve qu’un message a été émis ou reçu)
  • défiguration de site Web (defacement)
  • phishing
  • réseaux zombies ou botnets
  • exploitation de vulnérabilités
  • spam
  • scam
  • malware (virus, worm, trojan, spyware, adware, browser-hijacker)

Le déni de service

Il consiste à paralyser un réseau ou un site dont on ne parvient pas à en obtenir l'accès. Le déni de service distribué utilise plusieurs machines relais à partir d'une seule machine pour saturer un système.

Phishing

L’utilisateur est amené à cliquer sur un lien envoyé par email lui donnant accès à un site web ressemblant au site officiel, faisant ainsi croire à l’utilisateur qu’il navigue sur un site de confiance. Lorsque l'utilisateur tape ses informations confidentielles, celles-ci sont dérobées à son insu.

Les botnets

Il s'agit d'un ensemble de robots informatiques qui se déploient sur de grandes quantités de machines pour les utiliser comme relais de spam ou d'attaques.

Exploitation de vulnérabilités

L’exploitation de vulnérabilité consiste à abuser de failles connues dans les navigateurs, les applications web, les réseaux, les systèmes d'exploitation et les applications de la machine.

L'une des techniques d'exploitation de vulnérabilité les plus utilisées sur le Web est l'injection SQL, qui consiste à insérer dans un formulaire non protégé du code SQL capable d'altérer ou de détruire la base de données sur laquelle évolue le site Web attaqué.

Une autre technique de hacking sur le Web très utilisée est le Cross Site Scripting XSS, qui injecte du code côté client sur les pages Web visitées par d'autres utilisateurs, et permet de se connecter en tant qu'un autre utilisateur sans connaître le mot de passe de cet utilisateur.

Toutes les vulnérabilités recensées sont enregistrées dans le répertoire du CVE afin d'en conserver l'historique et le mode opératoire. Tout individu peut contribuer à cette base de données en publiant une nouvelle vulnérabilité.

Les Spam

Il s'agit d'une technique de marketing consistant à diffuser massivement par email de la publicité non sollicitée par les utilisateurs.

Le Scam

Les scams exploitent la naïveté des utilisateurs afin qu'ils fournissent de leur plein gré des données confidentilles.

Les virus

Les différentes contaminations possibles sont les suivantes :

  • par les supports amovibles
  • par les pièces jointes des emails
  • par les exécutables
  • par les fichiers multimedia mp3, mp4, avi, …

Les worms

Il s'agit d'un type d'attaque particulier qui utilise un carnet d'adresse pour se propager.

Les trojans

Il s'agit d'un virus qui s'insère dans une machine puis donne accès aux ressources de cette machine à l'insu de son utilisateur.

Les spyware

Il s'agit d'un terme général relatif à un malware qui récupère des informations à partir d'une machine infectée

Les adware

Il s'agit d'un logiciel qui présente des publicités non sollicités, souvent sous forme de fenêtre pop-up.

Les browser-hijackers

Il s’agit de modifications du navigateur sans la permission de l’utilisateur pour afficher des publicités ou enregistrer l’activité de l’utilisateur. Un browser-hijacker peut par exemple être utilisé pour génèrer de fausses alertes de sécurité afin d'inciter à acheter un anti-virus.
